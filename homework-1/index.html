Шпаргалка

  • git config --global user.name blabla - налаштування імені
  • git config --global user.email blabal@bla - налаштування email
  • git init - створює порожній репозиторій
  • git status - для перевірки статусу файлів
  • git add - додавання вмісту файлів
  • git commit - створює коміт, записує зміни до репозиторію
  • git log - показує історію комітів
  • git checkout - перемикає гілки або відновлює файли робочого дерева
  • git branch -створює або видаляє гілки, дозволяє переглядати, перейменовувати їх
  • git push - відправляє у віддалений репозиторій вміст локального репозиторію
  • git pull - викачує вміст віддаленого репозитрію, оновлює ним локальний
  • git merge - об'єднує зміни у поточну гілку
  • git clone - клонує репозиторій в нову директорію

.gitignore файл - визначає файли, які потрібно ігнорувати

Advanced

  • git revert - скасовує дії минулих комітів (не видаляє, а створює новий коміт з відміною непотрібних дій)
  • git reset - скасовує локальні зміни в репозиторії
  • git rebase - переписує історію (зміни) однієї гілки поверх іншої
  • git cherry-pick - переносить коміти з однієї гілки в іншу
  • git stash - відпраляє в архів зміни, які здійснені у файлах
